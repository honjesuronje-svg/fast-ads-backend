# Fix 404 Error - Step by Step

## The Problem
You're getting 404 because there's another Laravel application running on port 8000.

## Solution

### Step 1: Kill the existing server

```bash
# Find and kill process on port 8000
lsof -ti:8000 | xargs kill -9

# Or kill all artisan serve processes
pkill -f "php artisan serve"
```

### Step 2: Start server from correct directory

```bash
cd /home/lamkapro/fast-ads-backend/laravel-backend

# Clear caches
php artisan route:clear
php artisan config:clear

# Start server
php artisan serve --host=127.0.0.1 --port=8000
```

**IMPORTANT**: Keep this terminal open! The server must keep running.

### Step 3: Test in a NEW terminal

Open a **new terminal window** and run:

```bash
curl -H "X-API-Key: invalid_key" \
  -X POST http://127.0.0.1:8000/api/v1/ads/decision \
  -H "Content-Type: application/json" \
  -d '{"tenant_id": 1, "channel": "news", "ad_break_id": "test", "position": "pre-roll", "duration_seconds": 30}'
```

You should now get:
```json
{
  "success": false,
  "error": {
    "code": "INVALID_API_KEY",
    "message": "Invalid or inactive API key"
  }
}
```
